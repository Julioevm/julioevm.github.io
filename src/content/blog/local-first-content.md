# Local First Content

Local Markdown is a useful default for a personal site because it keeps authoring close to the code.

For this version, the blog and project entries are static assets. That gives the site predictable builds, simple hosting, and no dependency on live API limits.

Later, live GitHub data can be layered in for stars, recent commits, or pinned repositories. The content model already keeps external links separate from the page itself.
